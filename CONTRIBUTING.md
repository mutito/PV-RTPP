# Contributing to RTPP

Thanks for contributing.

## How to Contribute

1. Fork the repository.
2. Create a feature branch from main.
3. Make your changes.
4. Open a pull request back to this repository.

All modifications should be proposed through pull requests so changes can be reviewed and tracked.

## Pull Request Guidelines

- Keep each PR focused on one change.
- Use a clear title and description.
- Explain why the change is needed and what was tested.
- Update docs when interfaces, behavior, or deployment steps change.

## Scope Guidance

- Keep core library logic generic.
- Put site-specific mappings in RTPP_CALC/RTPP_PROGRAM.xml.
